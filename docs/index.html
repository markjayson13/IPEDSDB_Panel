What This Snapshot Does

The browser only lists columns that exist in the published cleaned panel snapshot. It helps you search, review, and export a usable variable list.
Discovery Search, facet-filter, browse semantic families, and inspect detailed metadata without leaving the page.
Validation Preview missing or duplicate imports before changing the current selection.
Export Copy or download selectedvars.txt and an optional manifest.

Quick Start

Search by concept, variable name, or component family.
Use filters to shrink the list before selecting in bulk.
Open details when you need descriptions, coverage, or related fields.
Export a plain-text variable list when you are done.
The separate User Guide walks through the workflow in a little more detail.

Dependability Notes

This browser is tied to a specific panel release, not a live backend.
Nonexistent columns are excluded because the list comes from the panel schema itself.
This landing page reads its release metadata from the published browser snapshot.
The browser stays fully static and self-contained, which keeps deployment simple on GitHub Pages.

Good To Know

UNITID and year remain retained keys in the custom-panel workflow.
The public browser snapshot works without a server because it is a single committed HTML artifact.
The export format remains intentionally simple: one variable per line.
This page describes the browser snapshot. It does not replace the underlying panel QA documentation.

Trust Boundaries

This browser is a variable-selection aid. It is not the source of truth for data values and it does not edit the parquet.
If the cleaned panel schema changes, the browser snapshot should be refreshed.
If dictionary metadata changes, labels or descriptions can become stale until the next publish.
The browser helps reduce selection mistakes. It does not certify research design quality.
GitHub Pages serves this site from the repository docs/ folder. The browser artifact itself is docs/variable-browser.html.
